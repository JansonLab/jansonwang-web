`top` 命令的输出是你诊断 Linux/Unix 系统性能的“仪表盘”。它分为两个主要部分：顶部的系统概览和下方的进程列表。

```shell
top - 17:15:29 up 12:36,  1 user,  load average: 1.24, 1.38, 0.84
任务: 556 total,   1 running, 555 sleeping,   0 stopped,   0 zombie
%Cpu(s):  0.3 us,  0.1 sy,  0.0 ni, 99.5 id,  0.0 wa,  0.0 hi,  0.0 si,  0.0 st
MiB Mem :   7894.5 total,    390.1 free,   6750.0 used,    754.4 buff/cache
MiB Swap:   2048.0 total,     12.3 free,   2035.7 used.    730.5 avail Mem

 进程号 USER      PR  NI    VIRT    RES    SHR    %CPU  %MEM     TIME+ COMMAND
   1889 ubuntu    20   0 5069800  92812  34196 S   3.0   1.1  17:29.78 gnome-shell
   2686 ubuntu    20   0  645292  23316  11020 S   1.0   0.3   0:09.09 gnome-terminal-
```


## 系统概述 {#系统概述}


### 第1行：任务队列信息 {#第1行-任务队列信息}

`top - 17:15:29 up 12:36,  1 user,  load average: 1.24, 1.38, 0.84`

-   `17:15:29` ：当前系统时间。
-   `up 12:36` ：系统已运行时间（从上次开机到现在）
-   `1 user` ：当前登陆到系统的用户。
-   `load average: 1.24, 1.38, 0.84` ：【相当重要】系统在过去 1 分钟、5 分钟、15 分钟内的平均负载。
    -   这个数字大致表示正在运行或等待 CPU 的进程数。
    -   在一个单核 CPU 上，1.00 意味着满负荷。在你的系统上（假设是多核，比如 4 核），1.24 意味着负载是轻到中度的。如果这个数字持续高于你的 CPU 核心数，说明 CPU 资源不足。


### 第2行：任务（进程） {#第2行-任务-进程}

`任务: 556 total,   1 running, 555 sleeping,   0 stopped,   0 zombie`

-   `556 total` ：系统当前的总进程数。
-   `1 running` ：正在运行或在运行队列中等待CPU的进程数。
-   `555 sleeping` ：正在“睡眠”的进程数。它们在等待某个时间发生（例如 I/O 操作、定时器）。
-   `0 stopped` ：被停止的进程（例如按了 Ctrl+Z）。
-   `0 zombie` ：僵尸进程。指已经执行完毕，但父进程尚未读取其退出状态的进程。


### 第3行：CPU状态 {#第3行-cpu状态}

`%Cpu(s):  0.3 us,  0.1 sy,  0.0 ni, 99.5 id,  0.0 wa,  0.0 hi,  0.0 si,  0.0 st`

这一行显示了 CPU 时间的分配百分比：

-   `us` （user）：用户空间进程占用的 CPU 百分比。
-   `sy` （system）：内核空间占用的 CPU 百分比。
-   `ni` （nice）：被调整过优先级(nice)的用户进程占用的 CPU 百分比。
-   `id` （idle）：[非常重要]CPU空间时间的百分比。此处的 `99.5 id` 表明 CPU 目前非常空闲。
-   `wa` （iowait）：CPU 等待 I/O 操作（如读写硬盘）完成的时间百分比。如果这个值很高，通常意味着硬盘是瓶颈。
-   `hi` （hardware irq）：处理硬件中断占用的 CPU 百分比。
-   `si` （software irq）：处理软件中断占用的 CPU 百分比。
-   `st` （steal time）：（仅适用于虚拟机）被宿主机(hypervisor)“偷走”的 CPU 时间。


### 第4行：物理内存（RAM） {#第4行-物理内存-ram}

`MiB Mem :   7894.5 total,    390.1 free,   6750.0 used,    754.4 buff/cache`

-   `total` ：总物理内存（约8GB）
-   `free` ：完全未被使用的内存。
-   `used` ：已被使用的内存。
-   `buff/cache` ：【重要概念】被 Linux 内核用作缓冲区 (buffers) 和页面缓存 (cache) 的内存。
    -   这是为了加速文件读写。Linux 会尽可能多地使用空闲内存来做缓存。
    -   这部分内存在需要时可以被立即释放给应用程序。


### 第5行：交换空间（Swap） {#第5行-交换空间-swap}

`MiB Swap: 2048.0 total, 12.3 free, 2035.7 used. 730.5 avail Mem`

-   `total` ：总交换空间大小（通常在硬盘上，约2GB）。
-   `free` ：可用的交换空间。
-   `used` ：已被使用的交换空间。当物理内存(RAM)不足时，系统会把不常用的内存页移动到 Swap 分区。
-   `avail Mem` ：【最重要】真正可用的内存。它约等于 `free` 内存 + 大部分的 `buff/cache` 内存。这是你衡量系统还剩多少内存的首要指标。


## 进程列表 {#进程列表}

这部分显示了当前消耗资源最多的进程。

`进程号 USER PR NI VIRT RES SHR S %CPU %MEM TIME+ COMMAND`

-   `进程号` (PID) ：进程的唯一ID。
-   `USER` ：运行该进程的用户。
-   `PR` ：进程的调度优先级（由内核动态调整）
-   `NI` ：Nice 值。负责表示高优先级，正值表示低优先级（默认为0）
-   `VIRT` (Virtual Memory)：进程使用的虚拟内存总量。这个值通常很大，不太能反映真实内存占用。
-   `RES` (Resident Memory)：【非常重要】进程当前实际占用的物理内存(RAM)大小。这是判断进程内存占用的关键指标。
-   `SHR` （Shared Memory）：进程使用的共享内存大小。
-   `S` (Status)：进程的状态。
    -   `S` = Sleeping(休眠中)
    -   `R` = Running(运行中)
    -   `D` = Uninterruptible Sleep(不可中断的休眠，通常在等待 I/O)
    -   `Z` = Zombie(僵尸进程)
    -   `T` = Stopped(已停止)

-   `%CPU` ： 自上次 `top` 刷新以来，进程占用的 CPU 百分比。
-   `%MEM` ： 进程占用的物理内存百分比（即 `RES` 占总内存的比例）。
-   `TIME+` ：进程自启动以来累计使用了 CPU 总时间（精确到百分之一秒）。
-   `COMMAND` ：进程的命令名或启动路径。


## 实例分析 {#实例分析}

对上述 `top` 命令的输出的分析如下：

1.  CPU 状态：良好。CPU 有 99.5% 的空闲 (id)，说明 CPU 完全没有压力。
2.  内存状态：严重不足！物理内存 (RAM) used 很高（6750.0 / 7894.5）。
    -   交换空间 (Swap) 几乎用满 (2035.7 / 2048.0)。
    -   avail Mem（真正可用）只剩 730.5 MiB。

3.  结论： 你的系统正在因为内存不足而大量使用 Swap。这会导致系统运行缓慢（因为硬盘读写远慢于内存），尽管你的 CPU 看起来很空闲（它可能在 wa 等待 I/O，虽然这次抓取时 wa 为 0）。

<!--listend-->

1.  罪魁祸首：
    -   `chrome` 进程 (PID 39811) 一个进程就占用了 2.6g 物理内存 (`RES` 列)，占了你总内存的 34.2% (`%MEM` 列)。
    -   另一个 `chrome` 进程 (PID 41331) 也占用了 435MB 内存。
    -   Chrome 浏览器是导致你内存耗尽和系统卡顿的主要原因。


## TOP 输出优化 {#top-输出优化}

在 `top` 命令中，你有两种主要的方式来按不同的列进行排序：

1.  交互模式（在 `top` 运行时按快捷键） - 这是最常用、最灵活的方式。
2.  命令行参数（启动 `top` 时指定） - 适用于在启动时就设定好排序。


### 交互模式（运行时排序） {#交互模式-运行时排序}

1.  常用快捷键（按一下立即生效）
    -   `M` (大写 M) : 按 内存使用率 ( `%MEM` ) 降序排序。
    -   `P` (大写 P) : 按 CPU使用率（ `%CPU` ）降序排序。
    -   `T` (大写 T) : 按累计运行时间 ( `TIME+`) 降序排序。
    -   `N` (大写 N) : 按进程号( `PID` ) 降序排序。

<!--listend-->

1.  切换排序方向
    -   ~R~(大写R) ： 反转 (Reverse) 当前的排序顺序（在“升序”和“降序”之间切换）。

<!--listend-->

1.  高亮和切换排序列
    -   `x` (小写x):高亮当前用于排序的列。
    -   `<` (小于号):向左移动排序列（例如，从 %MEM 切换到 %CPU）。
    -   `>` (大于号)：向右移动排序列（例如，从 %CPU 切换到 %MEM）。

<!--listend-->

1.  高级菜单（F键）这是最强大的方式，可以按任意列排序：
    -   在 top 运行时，按 f (小写 f) 键，会进入 "Fields Management" (字段管理) 界面。
    -   你会看到一个所有列的列表，当前排序列（比如 COMMAND）前面会标有一个星号 \*。
    -   使用上/下箭头来移动光标，选中你想要排序的列（例如，移动到 RES 实际内存）。
    -   按 s (小写 s) 键，将其设为 (Set) 排序字段。
    -   按 Esc 键或 q 键退出此界面，返回到主视图，你将看到进程已按你新选的列排序。


### 命令行参数（启动时排序） {#命令行参数-启动时排序}

你可以在启动 top 时就指定排序列。

-   使用 -o 标志（注意是小写 o）后跟你想排序的列名。
-   列名通常与你在 top 界面顶栏看到的完全一致。

示例：

-   按内存启动：
    ```shell
    top -o %MEM
    ```

<!--listend-->

-   按 PID 启动：
    ```shell
    top -o PID
    ```

<!--listend-->

-   按实际内存（RES）启动：
    ```shell
    top -o RES
    ```
